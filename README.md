# 对抗性数据改写在欺诈对话检测中的应用

**项目说明**：本项目基于实验一和实验二的成果，实现了对抗性数据改写技术在欺诈对话检测中的应用研究。

## 项目结构

```
├── exp1.py                    # 实验一：基于TF-IDF+SVM的诈骗检测
├── exp2.py                    # 实验二：对话行为与交互策略建模
├── adversarial_rewrite.py     # 对抗性改写核心实现
├── 实验结果展示.py            # 实验结果可视化展示
├── 大作业-对抗性数据改写在欺诈对话检测中的应用.md  # 完整大作业报告
├── attack_results.csv         # 攻击效果评估结果
├── behavior_strategy_analysis.csv  # 实验二行为策略分析结果
├── model_features.csv         # 实验二模型特征数据
├── adversarial_data_*.csv     # 各种攻击策略生成的对抗数据集
└── 通话数据互动策略结果/      # 原始数据集目录
```

## 实验概述

### 实验一：传统诈骗检测
- **方法**：TF-IDF + LinearSVC
- **性能**：测试集准确率76.1%，F1=0.796
- **文件**：`exp1.py`, `test_predictions.csv`

### 实验二：对话行为建模
- **方法**：对话行为识别（规则-based） + 交互策略分析 + 机器学习分类
- **性能**：准确率76.9%，F1=0.801
- **文件**：`exp2.py`, `behavior_strategy_analysis.csv`

### 实验三：对抗性改写攻击
- **方法**：四种改写策略（同义词替换、句式重构、模糊化表达、组合攻击）
- **评估**：在三种模型上测试攻击效果
- **文件**：`adversarial_rewrite.py`, `attack_results.csv`

## 核心发现

1. **模型鲁棒性评估**：传统机器学习模型对轻微文本扰动具有较强抵抗力
2. **攻击策略有效性**：验证了多种对抗改写技术的可行性
3. **语义保持挑战**：在攻击效果和语义完整性之间需要平衡
4. **应用前景**：为提高欺诈检测系统的安全性提供了新思路

## 技术特点

### 对抗改写策略
1. **同义词替换**：使用预定义同义词词典替换关键词
2. **句式重构**：改变句子结构但保持语义
3. **模糊化表达**：使用更委婉或模糊的表述
4. **组合攻击**：多种策略结合使用

### 评估指标
- **准确率下降**：改写前后模型准确率之差
- **攻击成功率**：成功改变模型预测的样本比例
- **语义保持度**：人工评估改写文本的语义完整性
